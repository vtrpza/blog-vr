# Checklist Anti-Slop — Revisão Humana

## Estrutura e variação
- [ ] Primeiro parágrafo: cena concreta ou pergunta real (não definição de dicionário)
- [ ] Resposta curta aparece nos primeiros 2 parágrafos
- [ ] Parágrafos com extensão variada (2-7 linhas, sem padrão)
- [ ] Pelo menos 3 frases curtas (≤8 palavras) intercaladas com ≥15 palavras
- [ ] Voz ativa >80% dos verbos (verbo ser/estar como exceção)

## Slop lexical (tolerância zero)
- [ ] Zero "Neste artigo", "Vamos explorar", "Descubra como"
- [ ] Zero "É importante ressaltar", "Vale destacar", "Não podemos deixar de mencionar"
- [ ] Zero "No mundo atual", "No cenário jurídico contemporâneo"
- [ ] Zero "complexo", "multifacetado", "crucial", "fundamental", "essencial" (contexto genérico)
- [ ] Máximo 2 em-dashes (—) no artigo inteiro
- [ ] Nenhuma variação de "em resumo" ou "concluindo"

## Conteúdo jurídico
- [ ] Toda afirmação jurídica tem fonte linkada
- [ ] Termo técnico explicado na primeira menção
- [ ] Nenhuma promessa de resultado ("vamos reduzir", "garantido", "solução")
- [ ] Nenhuma comparação com outros advogados/escritórios
- [ ] Disclaimer embutido em pelo menos 1 lugar além do `<Disclaimer />`

## CTA
- [ ] Linguagem informativa ("Solicitar análise", não "Contrate agora")
- [ ] 3 posições no artigo (após resposta curta, meio, final)
- [ ] Atributos data-track-cta presentes no HTML

## Teste final
- [ ] Leitura em voz alta soa natural
- [ ] Um leigo entenderia sem google
- [ ] Um advogado não apontaria erro factual
